V680-CHUD 读写工具
===================

本工具使用 Python + Tkinter，通过 USB 虚拟串口与 **OMRON V680‑CHUD** 通信，对 **V680‑D8KF67** 标签进行读写。  
串口通信基于你确认的 **RDA1 / WTA1** 命令格式实现，并支持自定义读取字节数（默认 26 字节）。

环境与安装
----------

1. 确保已安装 Python 3.7+。
2. 安装依赖：

   ```bash
   pip install pyserial
   ```

3. 将本仓库放在：

   ```text
   c:\Users\ychen\Desktop\680s
   ```

4. 运行程序：

   ```bash
   cd c:\Users\ychen\Desktop\680s
   python v680_chud_app.py
   ```

串口协议说明（RDA1 / WTA1）
---------------------------

- **读取命令**  
  格式：

  ```text
  RDA1 + AAAA + NN + 终结符
  ```

  - `AAAA`：4 位十六进制地址（字节地址，如 `0000`）
  - `NN`：2 位 **十进制** 长度（01~99，表示字节数）  
  - 终结符：可选 `*` / `*CR` / `CR`（默认使用 `*CR`，即 `*\r`）

  示例（从地址 0 读取 20 字节）：

  ```text
  RDA1000020*\\r
  ```

- **写入命令**

  ```text
  WTA1 + AAAA + DATA + 终结符
  ```

  - `AAAA`：4 位十六进制起始地址
  - `DATA`：要写入的 ASCII 文本（程序会将非可见字符替换为 `.` 发送）

  示例（从地址 0 写入 `HELLO`）：

  ```text
  WTA10000HELLO*\\r
  ```

- **读响应（典型）**

  ```text
  RD00 + DATA + EE + *
  ```

  - `DATA`：标签返回的数据（ASCII）
  - `EE`：2 位十六进制结束码
    - `00` 或 `90` 被视为成功（本工具会同时接受这两种）
  - 工具只截取 `RD00` 之后、`EE` 之前的部分作为有效数据。

界面使用说明
------------

启动后界面包含以下区域：

1. **USB(COM 口)**  
   - 选择系统中显示的 `COM` 口和波特率（一般为 9600）。  
   - 点击“连接”建立/断开与 V680‑CHUD 的连接。

2. **芯片信息**  
   - `读取芯片信息`：从地址 0 读取指定长度的数据并展示。  
   - `长度(字节)`：要读取的字节数，默认 `26`。可根据需要修改，例如 20、32 等。  
   - 下方显示对应的十六进制与 ASCII 字符串，同时在大文本区域也会显示一份。

3. **读写 V680‑D8KF67**  
   - `起始地址(字节)`：0~8191。  
   - `长度(字节)`：本次要读取的长度。  
   - `读取`：从起始地址读取指定长度并在下方文本框中以“十六进制+ASCII”与“仅 ASCII”两种形式显示。  
   - `读取全部(8KB)`：从 0 开始读取 8192 字节（注意耗时与输出长度）。  
   - `写入下方内容`：将“写入内容(ASCII)”区域的文本按当前起始地址写入标签。

4. **写入内容(ASCII)**  
   - 在此输入要写入到标签的数据（仅 ASCII 文本）。  
   - 写入前需确认起始地址与长度不会越界（程序也会检查）。

5. **调试 – 最近一次收发**  
   - `发送`：显示最近一次串口发送的完整命令（会将 `\r`、`\n` 显示为 `\\r`、`\\n`）。  
   - `接收`：显示最近一次从设备收到的完整响应（若为非 ASCII，会以 `hex: ...` 显示）。  
   - `测试连接 (TS)`：发送简单的 TEST 命令确认 PC 与 V680‑CHUD 之间的串口连接是否正常。  
   - `终结符`：可在 `*CR`、`*`、`CR` 之间切换，分别对应 `*\\r`、`*`、`\\r` 三种结尾。

错误处理与排查建议
------------------

- 若 **测试连接失败**：  
  - 检查是否选择了正确的 COM 口和波特率；  
  - 检查 Windows 设备管理器中是否能看到 V680‑CHUD 作为串口设备；  
  - 查看“最近一次收发”中的“接收”内容，并对照设备手册确认格式。

- 若 **读取失败**：  
  - 弹框会显示结束码与原始响应片段，例如 `结束码=72` 通常表示“标签未检测到”；  
  - 确认标签距离天线在规定范围内，再重试；  
  - 检查“发送”是否为期望的 RDA1 命令。

- 若 **写入失败**：  
  - 结束码为 `7D` 多为“写保护”错误，需参考设备手册取消写保护后再试；  
  - 其它错误码可以在手册的 End Code 表格中对照。

如需调整协议（例如设备使用不同的命令码或结束码），只需要修改 `v680_chud_app.py` 中的
`V680Protocol` 类，实现新的 `read_block` / `write_block` 与 `_parse_rd_response` 逻辑即可。
